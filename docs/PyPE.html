PyPE - Python Parsing Expressions Library (API)¶

The Pattern Base Class¶

Pattern is an abstract base class for all pattern objects. The most important function for patterns is match(string[, index]), which checks a string starting at the given index to see if it matches the pattern. A Match object is returned if a match is found, or None if the pattern did not match the string at the given index. Each subclass of Pattern implements different search patterns. Note that for convenience the pattern object can be called as if it is a function, and the call is forwarded to match(). Therefore, if ptn is a pattern, the following are equivalent:

>>> match = ptn.match("This is a test")
>>> match = ptn("This is a test")

A Match object allows you to get the string that was matched and any captures. It also includes a context value that provides access to any Stack objects that were created. Some patterns may succeed without consuming any values from the string. Some care must be taken to ensure that no recursive expressions occur that consume no input to avoid infinite loops. For reference, this is referred to as “left recursion” in PEG literature, and grammars can be rewritten to remove “left recursion”. Information on this can be found via an internet search.

Any pattern can be ‘named’ by calling setName for the pattern or by using the ‘set name’ operator (shown later). This is primarily used for debugging purposes or when building a Tokenizer, since all token patterns must have a name specified.

Debug mode can be set for any pattern by calling the function debug with True (to debug all subpatterns), False (to supress debug message for a pattern and subpatterns]), and "named" (to show only named patterns in the debug output).

Pattern¶

class PyPE.PyPE.Pattern¶

Abstract base class for all parsing expression patterns.

debug(debugOpt=None, **args)¶

Set or return the debug options for this pattern. The basic options that can be set are:

  • True or ‘show’ to show parsing debug information for the current pattern and all contained patterns.

  • False or ‘hide’ to suppress debug information for the current pattern and all contained patterns.

  • ‘named’ to show parsing debug information for any named patterns contained within the current pattern.

Other options may be added. For the available options, see the DebugOptions class constructor which is called with the parameters sent to this function.

When the debug option is set to True, pattern results are printed to the console by default. The output source can be modified via a custom DebugOptions object.

When the debug option is set to ‘show’, ‘hide’, or ‘named’, the option applies to any contained patterns that do not have some other other explicitly set.

Parameters:

  • debugOpt – This option can be an object that implements the beforeMatch and afterMatch functions with signatures that match the DebugOptions functions of the same name. If this is True or ‘show’, debugging is enabled. If this is False or ‘hide’, debugging is disabled. If this is ‘named’, only named patterns are shown. If this is ‘show_only_success’, only successful matches are shown. If this is a function, it is used as a DebugOptions afterMatch filter.

  • **args – If more control is needed, arguments can be passed in that are forwarded to the DebugOptions constructor. See the constructor for more details.

Returns:

When no options are passed in, this returns the debug options object for this Pattern. If parameters are passed in, a DebugOptions object is created and associated with this Pattern.

The debug option can be set using:

>>> p = P(1).debug(True)
>>> p = P(1) & 'show'  # Same as True
>>> p = 'hide' & P(1)  # Same as False
match(string, index=0, context=None)¶

Match the Pattern against the string starting at the given index. Note that Pattern implements the __call__ method which forwards calls to this function.

Parameters:

  • string – A string to match.

  • index – The location in the string to look for the given pattern.

  • context – Store stack information and information that is passed forward during match operations.

Returns:

A Match object if the pattern succeeds, or None if the pattern fails.

>>> p = P('t')   # Match the 't' character
>>> p.match('test')
t
>>> p('test')  # Equivalent to call above
t
setName(name)¶

Set the name for a pattern object. When debugging, the name is printed in the debugging output in place of the Pattern specification. It is also used by the Tokenizer class.

Parameters:

name – The name for the Pattern.

Returns:

The Pattern object. Allows chaining of commands.

>>> p = P(1).setName('test 1')
>>> p.name
'test 1'
>>> p = 'test 2' | P(1)
>>> p.name
'test 2'
>>> p = P(1) | 'test 3'
>>> p.name
'test 3'

match_TF(string, index=0, context=None)¶

Return False if the value is False, or the ‘index’ value if not past end of string.

>>> p = P(False)
>>> p("test") is None
True
>>> p = P(True)
>>> p("test") == ""
True

match_fn(string, index=0, context=None)¶

Call the function with signature:

match = fn(string, index, context)

The function can return one of the following:

  • A match object

  • An integer between index and the length of the string. This results in a match object ending at the given integer.

  • True results in a match object beginning and ending at the current index

  • False or any value not matching the above results in a match value of None.

>>> p = P(lambda str, i, ctxt: i+2)
>>> p("test")
te
>>> p("test",3) is None
True
>>> p = P(lambda str, i, ctxt: True)
>>> p("test",4) == ""
True
Parameters:

  • string – The string to match

  • index – The location in string to start match

  • context – Information that is forwarded between matches.

match_n(string, index=0, context=None)¶

Matches exactly n characters.

>>> p = P(1)

>>> p("abc")

>>> p("abc",1) is None
True
Parameters:

  • context – Information that is forwarded between matches.

match_neg(string, index=0, context=None)¶

Matches less than n characters only (end of string).

>>> p = P(-3)

Parameters:

Matches the pattern passed in.

Cs (Stack Capture)¶

class PyPE.Cs(stack, n=1, pop_from_stack=False)¶

Capture values from a stack

match(string, index=0, context=None)¶

Capture values from a stack

Parameters:

  • string – The string to match against.

  • index – The current location in the string.

  • context – Information that is forwarded between matches.

Returns:

A match object stack values added as captures.

Examples:

>>> p = Sc('test',Cc('1')*Cc('2')*Cc('3')) * Cs('test')
>>> match = p.match("")
>>> match.captures
['3']
>>> match.context.getStack('test')[0:]
['1', '2', '3']

>>> p = Sc('test',Cc('1')*Cc('2')*Cc('3')) * Cs('test', 2, pop_from_stack=True)
>>> match = p.match("")
>>> match.captures
['2', '3']
>>> match.context.getStack('test')[0:]
['1']

Match Objects¶

A Match object is returned when the match(string, index) function for a pattern is called with a string, and the string matches the pattern at the given index. The Match object indicates the portion of the string that was matched, the start and end location of the match, and contains any captures from the pattern.

Match¶

class PyPE.PyPE.Match(string, start, end=None)¶

The Match object tracks the text that was matched by a PEG pattern, and provides access to any match captures.

Variables:

  • string – The original string that was matched.

  • start – The location in the string where the pattern match starts.

  • end – The location in the string where the pattern match ends.

  • captures – The Pattern captures. Note that the match.getCapture(n) method is available to access captures, or simply match[n]. len(match) indicates the number of captures.

getCapture(index)¶

Get the capture at the specified index. If the index does not exist, throw an IndexError. Note that the number of captures for a Match object named match is given by len(match). Note that array syntax can be used with the Match object as a shorthand for calling this method. For example, match[1] is equivalent to match.getCapture(1).

Parameters:

index – The index of the capture with the first capture being index 0. A negative index is used to access values relative to the end of the array, with index -1 being the last capture value.

Returns:

The requested capture value.

getValue(default=None)¶

Get the string that was matched by the PEG pattern.

hasCaptures()¶
Returns:

Return true if this Match object contains any captures.
